Client-Server Architecture

Definition

Client-Server architecture is a network model where a client requests services or resources and the server provides them. The client sends a request and the server processes the request and returns a response.

Components of Client-Server Architecture

1. Client

The client is the user's device or browser that sends requests to the server. Examples include Google Chrome, Firefox, and mobile browsers.

2. Server

The server is a powerful computer that stores websites, applications, and data. It processes requests sent by the client and sends responses back.

3. Internet

The internet acts as the communication medium between the client and the server. Data travels through protocols such as HTTP and HTTPS.

What Happens When You Open a Website?

  1. The user enters a website URL in the browser.
  2. The browser sends a DNS request to find the server IP address.
  3. The browser sends an HTTP request to the web server.
  4. The server processes the request.
  5. The server sends an HTTP response containing HTML, CSS, and JavaScript.
  6. The browser renders the webpage and displays it to the user.

Simple Client-Server Flow

Client (Browser) → Internet → Server → Internet → Client → Webpage Displayed
